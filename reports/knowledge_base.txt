FILE_NAME = "expenses.txt"

def add_expense():
    date = input("Enter date (YYYY-MM-DD): ").strip()
    description = input("Enter description: ").strip()
    amount = input("Enter amount: ").strip()

    # Save to file
    with open(FILE_NAME, "a") as f:
        f.write(f"{date},{description},{amount}\n")

    print(" Expense added successfully!")

def view_expenses():
    try:
        with open(FILE_NAME, "r") as f:
            lines = f.readlines()

        if not lines:
            print("No expenses recorded yet.")
            return

        print("\n Expense List:")
        print("-" * 40)
        total = 0
        for line in lines:
            date, description, amount = line.strip().split(",")
            print(f"{date} | {description} | ₹{amount}")
            total += float(amount)
        print("-" * 40)
        print(f"Total: ₹{total:.2f}")

    except FileNotFoundError:
        print("No expenses found. Add some first!")

def main():
    while True:
        print("\n==== Expense Tracker ====")
        print("1. Add Expense")
        print("2. View Expenses")
        print("3. Exit")
        choice = input("Choose an option: ").strip()

        if choice == "1":
            add_expense()
        elif choice == "2":
            view_expenses()
        elif choice == "3":
            print("Goodbye!")
            break
        else:
            print("Invalid choice, try again.")

if __name__ == "__main__":
    main()

# Contact Book - Save, Search, View Contacts
FILE_NAME = "contacts.txt"

def add_contact():
    name = input("Enter name: ").strip()
    phone = input("Enter phone number: ").strip()
    email = input("Enter email: ").strip()

    with open(FILE_NAME, "a") as f:
        f.write(f"{name},{phone},{email}\n")

    print(" Contact added successfully!")

def view_contacts():
    try:
        with open(FILE_NAME, "r") as f:
            lines = f.readlines()

        if not lines:
            print("No contacts found.")
            return

        print("\n Contact List:")
        print("-" * 40)
        for line in lines:
            name, phone, email = line.strip().split(",")
            print(f"Name: {name} | Phone: {phone} | Email: {email}")
        print("-" * 40)

    except FileNotFoundError:
        print("No contacts found. Add some first!")

def search_contacts():
    keyword = input("Enter name or phone to search: ").strip().lower()
    found = False

    try:
        with open(FILE_NAME, "r") as f:
            for line in f:
                name, phone, email = line.strip().split(",")
                if keyword in name.lower() or keyword in phone:
                    print(f"\n Match Found:")
                    print(f"Name: {name} | Phone: {phone} | Email: {email}")
                    found = True

        if not found:
            print("No matching contact found.")

    except FileNotFoundError:
        print("No contacts found. Add some first!")

    while True:
        print("\n==== Contact Book ====")

            view_contacts()

            search_contacts()
        elif choice == "4":

            print("Invalid choice, try again.")

if __name__ == "__main__":

Traditional fact-checking relies on human expertise and manual verification processes, approaches that are fundamentally limited in scalability when confronted with the exponential growth of digital content. LITERATURE SURVEY

2.1 Overview
The field of fake news detection and misinformation identification has emerged as a significant research area within natural language processing and computational social science over the past decade. Early approaches to identifying misinformation focused on analyzing source credibility, author reputation, and network patterns. However, as the sophistication of misinformation increased, research shifted toward lingu. The research landscape encompasses several related subfields: veracity prediction (predicting the truthfulness of claims), rumor detection (identifying false or unverified information on social media), stance detection (determining the position of text relative to claims), and claim verification (assessing whether evidence supports specific claims). The methods developed in these related areas often share common preprocessing techniques, feature extraction strategies, and machine learning algorithms, though they may differ in their specific objectives and evaluation metrics. Key trends in the field include the increasing use istic and content-based analysis methods. Contemporary approaches leverage advances in deep learning, transformer-based models, and ensemble techniques to achieve higher detection accuracy of deep learning approaches, particularly recurrent neural networks (RNNs), convolutional neural networks (CNNs), and more recently, transformer-based models like BERT. There is also growing recognition of the importance of cross-domain validation, as models trained on one type of misinformation may not transfer well to other domains. Additionally, researchers increasingly acknowledge that perfect accuracy may be unrealistic; instead, the focus has shifted to understanding which types of false statements are detectable and which remain challenging, and to developing systems that augment rather than replace human fact-checkers
2.2 Existing Models and Frameworks

Comparative Analysis of Existing Approaches: The reviewed literature reveals several important patterns in the evolution of fake news detection methods. Early approaches relied primarily on traditional machine learning algorithms (Support Vector Machines, Logistic Regression) combined with hand-crafted linguistic features. These methods, while interpretable and computationally efficient, were limited by the quality of manual feature engineering. Subsequent developments introduced automated feature learning through deep neural networks, which consistently achieved higher accuracy but at the cost of reduced interpretability. The LIAR dataset, which is used in this project, emerged as a crucial benchmark in the field, established by Wang et al. (2017). This dataset contains 12,800 short statements from public figures labeled with verified veracity labels on a 6-point scale, which is commonly collapsed to binary labels (false vs. true) for simplified classification tasks. The availability of this standardized benchmark enabled comparative evaluation across different methodologies. Deep learning approaches, particularly bidirectional LSTM networks and transformer-based models, have demonstrated superior performance compared to traditional machine learning methods. The integration of attention mechanisms and domain-adversarial training further improved robustness across different sources and domains. However, these advances come with increased computational requirements and reduced interpretability compared to simpler models.
2.3 Limitations Identified from Literature Survey (Research Gaps)
Despite significant progress in fake news detection, several important research gaps and limitations persist:
Limited Transferability: Models trained on one domain or type of misinformation often fail to generalize well to other domains. A model trained on political statements may not perform well on health-related claims. This suggests that misinformation patterns may be domain-specific, and that robust detection requires either domain-adapted models or very large training sets covering diverse content.
Explainability and Interpretability: While deep learning models achieve higher accuracy, they provide limited insights into which specific features and patterns lead to false news identification. For practical fact-checking applications, understanding the reasoning behind classifications is often as important as accuracy itself. This gap between performance and interpretability motivates the continued development of more interpretable models.
Class Imbalance and Rare False Statements: In many real-world scenarios, false statements are less common than true ones, creating class imbalance problems. Additionally, novel types of misinformation may not be well-represented in training data. Models trained on existing false news may fail to detect emerging false statements with new characteristics
Temporal Dynamics: Most research treats statements as static snapshots. However, the veracity of statements may depend on temporal context—what is false at one time may be true later (or vice versa). Understanding how statement veracity changes over time and how models should be updated with new information remains an open problem.
Linguistic Sophistication of Modern Misinformation: As detection systems improve, those creating misinformation adapt their techniques to evade detection. The adversarial nature of the problem means that solutions must constantly evolve to address new obfuscation and manipulation techniques.

The project titled “Fake News Detection Using Machine Learning” successfully achieved its goal of automatically classifying news statements as true or false using natural language processing and supervised machine learning techniques.
A complete end-to-end pipeline was developed, starting from text preprocessing and feature extraction to model training, evaluation, and visualization. The LIAR dataset—containing 12,514 fact-checked statements—was used to train and validate multiple models, including Logistic Regression, Decision Tree, Gradient Boosting, and Random Forest.
The system performs three main functions effectively:
Text Preprocessing and Vectorization – Raw textual data is cleaned, tokenized, and transformed into numerical features using TF-IDF with unigram and bigram representations.
Model Training and Prediction – The processed text is passed through trained classification models to predict whether a statement is true or false.
Performance Analysis and Visualization – Multiple evaluation metrics (accuracy, precision, recall, F1-score) and visual tools (ROC curves, confusion matrices, and feature importance charts) are generated to analyze model behavior.
The final implementation demonstrated that traditional machine learning models, when combined with well-engineered text features, can effectively detect misinformation. The system achieved a maximum accuracy of 74.51% on unseen test data, confirming its robustness and real-world applicability.

4.1.1 Performance Evaluation
The performance of each model was evaluated using both validation and test datasets. Logistic Regression emerged as the most balanced model in terms of accuracy, interpretability, and stability, while Gradient Boosting achieved the highest numerical accuracy.








The Logistic Regression model displayed excellent recall (97%) for true statements and moderate performance for false statements (13% recall), reflecting the dataset’s natural imbalance. Gradient Boosting slightly outperformed others in accuracy but required longer training times.
Feature importance analysis revealed that linguistic patterns and specific word usage were strong predictors of veracity, aligning with known stylistic tendencies in false statements.
4.1.2 Comparisons
To evaluate effectiveness, the proposed system was compared against previous studies and common approaches from literature.

While advanced transformer-based models like BERT achieve slightly higher accuracy, they require significant computational power and lack transparency.
The proposed models strike an optimal balance between accuracy, speed, and explainability, making them well-suited for practical deployment on standard computing systems.
4.1.3 Testing Results
The testing phase assessed model generalization and prediction reliability using the held-out test set and additional evaluation visualizations
The model demonstrated stable generalization across the test set, consistent accuracy levels, and effective feature interpretation. Minor limitations were observed in detecting minority (false) classes due to dataset imbalance, but overall, performance was strong and reproducible
  


 4.2 Discussion
The proposed fake news detection system effectively integrates natural language processing and machine learning for automated misinformation detection. The workflow—combining TF-IDF vectorization with classical classifiers—proved both efficient and interpretable.

Key observations during experimentation include:
TF-IDF features with unigrams and bigrams significantly enhanced model accuracy and feature interpretability.
Logistic Regression provided nearly the same accuracy as Gradient Boosting, confirming the strength of linear models for textual data.
Class imbalance influenced recall for false statements, suggesting future use of resampling or class weighting.
Feature importance visualization revealed that exaggerated or emotionally charged words frequently indicated false statements.
The system’s results show that fake news detection can be achieved with high reliability using explainable ML techniques, even without deep neural networks.
This project highlights the practicality of traditional models in real-world misinformation screening, offering a foundation for future enhancements such as deep learning integration, real-time web monitoring, and cross-domain generalization.
